App spec — Execution Edge

Private single-user desktop/web app. Local data. No sharing.
1. Purpose
Separate calling quality (windows + leaders) from execution quality (deployment + sizing) so progress on the weak axis is visible and measurable over months.
2. Users
Single user (owner). No multi-tenant. No sharing. No leaderboards.
3. Privacy & data
All data stays on the user's machine. No cloud sync, no telemetry. Kite API credentials in OS keychain only. Local SQLite + CSV on disk; rely on OS-level full-disk encryption.
4. Data sources
Zerodha Kite Tradebook CSV (primary), Holdings CSV (for capital base), optional Kite Connect API with user's own key, user-entered Windows log.
5. Core entities
Window, Leader, Trade, Position, AccountSnapshot, MonthlyScorecard. Window has 0..N Leaders (identified), 0..N Trades (executed). Each trade may reference a Window and optionally a Leader.
6. Screens
Home, Windows, Capital Deployment, Position Sizing, Monthly Scorecard, Settings. Every KPI is shown with its trend, denominator, and thresholds on hover.
7. KPI definitions (frozen)
  • CDR = Σ deployed / capital available per window
  • Leader Coverage = leaders traded / leaders identified
  • Window Hit Rate = % windows where leader median beat benchmark by ≥ 2pp
  • Leader Edge = median(leader return) − benchmark return
  • Size–Conviction Correlation = Pearson corr(conviction tag, position % of capital)
  • Execution Gap = (target_deployment − actual_deployment) × leader_median_return
  • Execution Score = 100 × (0.4·CDR + 0.3·Coverage + 0.2·SizeDisc + 0.1·(1 − Gap/5%))
  • Deployment Velocity = days from window start to reaching 80% of peak deployment
8. Computation
Local Python service (FastAPI) + SQLite + pandas. All KPIs recomputed idempotently from raw Windows + Trades on each refresh.
9. UX rules
Never show a KPI without its 3-month trend. Never show a ratio without its denominator. Every KPI has a Good / Watch threshold on hover and a drill-through to the rows that produced it.
10. Inputs to log per window
Start, end, regime/thesis, conviction (1–5), list of leader symbols, notes. Everything else is derived from Trades.
11. Acceptance criteria
On ≥ 12 windows, renders all six screens in < 2s, recomputes KPIs deterministically, exports a 1-page monthly PDF review.
12. Non-goals
No signal generation or stock picking. No brokerage/tax attribution (use Zerodha Console). No social features.
13. Behavioural guardrails (opt-in)
Pre-trade checklist: "Is this inside a logged window? Conviction tag set? Position size within ±2pp of ladder?" Logged, never enforced.
14. Roadmap v2
Kite Connect auto-sync, intraday execution view, NIFTY / NIFTYMIDCAP overlay on window P&L, what-if simulator for "P&L at target CDR".

Privacy

This data is yours. Keep it that way.

⚠ Confidential — shadabkhaninv@gmail.com. Do not share.

  • All calculations in this mockup run locally in your browser. No network calls are made with your trading data.
  • In the production app, credentials (Kite API key/secret, access token) are stored in the OS keychain and never written to plain files.
  • The SQLite database and any CSV exports live in a folder you choose. Recommended: a folder on an encrypted volume (FileVault / BitLocker / LUKS).
  • No telemetry. No crash reporting. No third-party analytics.
  • You can wipe everything by deleting the app's data folder; there is no server-side copy.
  • Any cloud backup (iCloud / OneDrive / GDrive) is opt-in and off by default.
